Implementation Notes
Backend (minimal): Add power and stat_affected to InventoryItemSerializer.serialize() in src/api/serializers/inventory.py. Use a lookup dict to map item class names → stat strings ("hp", "fatigue", "status"). Derive heal_range as [int(power * 0.8), int(power * 1.2)].
Frontend: Modify the ally picker section of ItemDetailDialog.jsx (lines 785–819). Compute projectedPct = Math.min(100, (member.hp + item.effect.power) / member.max_hp * 100). Render two sibling <div>s inside .bar-track: the current fill at hpPct% and the gold projected segment at (projectedPct - hpPct)%.
Stat routing: If item.effect.stat_affected === "fatigue", replace the HP row with a FAT row (cyan label, cyan fill). Hide HP bar entirely for non-HP items.
Fallback: If item.effect is undefined/null, skip projected rendering entirely — falls back to State 1 behavior with no visual regression.
